# 房东收租管理系统 · 全套开发+推广方案（可直接落地）

# 第一部分：基础产品方案（回顾，衔接后续技术内容）

## 一、产品定位与核心信息

定位：轻量、无租客端、纯房东后台管理，简单、算得准、能对账、不扯皮，中年房东一键上手。

适用人群：个人房东（1～10套房）、二房东（10～50套房）、小型公寓托管

核心价值：自动算房租+水电+杂费（零出错）、自动催租/到期提醒、账单清晰减纠纷、财务可追溯、不接入支付（零合规风险）

付费模式：免费版（最多3套房）、标准版（69元/年，1～20套）、专业版（129元/年，不限套房）

## 二、核心缴费方式（关键）

系统不做支付通道、不碰资金，规避合规风险，全部走线下/外部转账（微信转账、支付宝转账、银行卡转账、现金）。

系统核心作用：1. 生成清晰账单；2. 房东手动标记“已缴费/未缴费”，满足房东“安全、不被抽成、不经过第三方”的核心诉求。

## 三、核心功能模块总结构

1. 房源管理；2. 租客管理；3. 账单管理（核心）；4. 抄表管理；5. 催租与提醒；6. 合同与到期管理；7. 财务统计；8. 系统设置

## 四、核心业务流程图（严谨版）

1. 出租流程：新增房源 → 添加租客并绑定 → 生成首次账单 → 租客转账 → 房东标记已缴费

2. 每月收租流程：抄表 → 新建账单（自动计算）→ 发账单给租客 → 租客转账 → 标记已缴费 → 计入财务

3. 退房流程：退房结算 → 计算应退押金 → 记录押金退还 → 房源改为空置

# 第三部分：前端页面原型结构（文字版，可直接对接前端开发）

说明：页面采用“顶部导航栏+左侧侧边栏+中间内容区”布局，适配PC端（房东主要用电脑操作），风格简洁、按钮醒目，避免复杂交互，适配中年房东操作习惯。所有页面均包含“返回”“保存”按钮，关键操作有确认弹窗。

## 一、公共组件（所有页面通用）

1. 顶部导航栏：左侧logo（可简化为“房东收租管理系统”文字）、右侧“个人中心”“退出登录”“帮助中心”按钮

2. 左侧侧边栏：所有功能模块入口（仪表盘、房源管理、租客管理、账单管理、抄表管理、催租提醒、合同管理、财务统计、系统设置），当前页面入口标红高亮

3. 弹窗组件：确认弹窗（如“确认删除？”“确认标记已缴费？”）、表单弹窗（如“标记已缴费”弹窗）

4. 搜索组件：所有列表页顶部均有搜索框，支持关键字搜索（如房源搜索支持房号、小区名称；租客搜索支持姓名、手机号）

5. 分页组件：所有列表页底部均有分页（默认每页10条，支持切换5/10/20条）

## 二、逐页详细原型（含交互逻辑）

### 1. 登录/注册页

布局：居中卡片式，背景简洁，无多余装饰

元素：

- 标题：房东收租管理系统

- 登录表单：账号输入框（提示“手机号/邮箱”）、密码输入框（带隐藏/显示密码按钮）、“记住密码”复选框、“忘记密码”链接、“登录”按钮（宽屏满铺，蓝色底色）

- 注册入口：底部“还没有账号？立即注册”链接

- 快捷登录：微信快捷登录按钮（可选，放登录按钮下方）

交互：

- 账号/密码为空时，点击登录按钮提示“请输入账号/密码”

- 账号密码错误时，提示“账号或密码错误，请重新输入”

- 点击“忘记密码”→ 跳转忘记密码页（输入手机号、获取验证码、重置密码）

- 点击“立即注册”→ 跳转注册页（输入账号、密码、确认密码、手机号、验证码）

- 登录成功 → 跳转仪表盘页面
